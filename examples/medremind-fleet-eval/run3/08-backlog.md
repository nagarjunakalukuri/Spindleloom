## How to read this backlog

- **Priority = MoSCoW carried from the PRD.** `Must` = inside the 3-sprint vertical slice (BG-004); `Should` = target release, deferred past the slice, not dropped; `Could` = opportunistic; `Won't` = §What we're not doing (PRD).
- **Every PBI is a vertical slice** unless it is explicitly typed a platform **enabler** (`PLAT` epic). Per the `backlog-manager` platform-extension exception + split-on-blocker rule, a shared backend component that several stories consume — here the **Classification Gate**, the **idempotency ledger**, the **PHI-minimization composer**, the **audit/consent store**, and the **BFF authZ layer** — is a separately-valuable enabler PBI, scheduled ahead of the UI/functional PBIs that depend on it. The SDD (`07-sdd.md` §Component breakdown) ratifies each of these as a real component (SDD-CMP-001…012), so this is not inventing horizontal layers — the seams already exist in the design.
- **Size** is a rough S/M/L hint only (ASSUMPTION-23); **story points are owned by `estimation-facilitator`**, not fixed here.
- **Sprint mapping** below is a dependency-ordered *proposal* (ASSUMPTION-24); final sprint assignment is `sprint-planner`'s to ratify.

## Notes

- **Enabler-first, not layer-first.** PBI-PLAT-001…-006 look like a horizontal layer but are the SDD's ratified shared components (SDD-CMP-001/-002/-003/-010/-011, SDD-NFR-003), each independently valuable (the safety guarantee, the audit trail, the PHI boundary) and consumed by multiple vertical stories — the platform-extension / split-on-blocker exception, applied deliberately. Without them the CTL and PHI guardrails (KPI-5) would be re-implemented per-story and drift.
- **CTL overlap is intentional.** PBI-CTL-002 (patient surface) and PBI-QUE-002 (pharmacist queue surface) both assert "no one-tap for Schedule II–V" on *different* surfaces; both trace to FRD-CTL-002 / FRD-QUE-002 and are kept separate so each surface has its own testable guard, matching the SDD's "no reachable automated state" invariant (SDD-SEC-001, SDD-CMP-012).
- **FRD-CTL-003 is realized by the enabler**, not a standalone functional PBI: its fail-closed *read* behavior is PBI-PLAT-001 (the gate); the consumption guards are PBI-CTL-001/-002 and PBI-RFL-003. This avoids duplicating the same obligation as both an enabler and a story.
