## Notes (splits, altitude calls, routing)

- **One enabler minted (PBI-ENABLE-001).** No `frd-writer`-seeded PBI carried the Twilio HIPAA BAA, yet SR-PHI-003 makes an executed BAA a production go-live gate for SMS and `srs-writer` flagged it as a possible paid-edition cost item. Per the agent contract's backend-readiness / split-on-blocker rule it is surfaced as a blocking enabler (Type Spike: confirm HIPAA eligibility + edition/cost, then execute), ranked first for lead time, blocking the SMS path in PBI-RMD-001 / PBI-RMD-002 / PBI-PHI-001. Routed to PM/compliance, not the code build flow.
- **Audit, observability, encryption, session, pacing are NOT separate PBIs — deliberately.** SR-AUD-*, SR-OBS-*, SR-SEC-001/002/005, SR-INT-001 are cross-cutting constraints realised *inside* the functional PBIs' design and folded into their Definition of Done (via the DOC-002 RFC sections SDD-CMP-008/012, SDD-SEC-001/003), not minted as horizontal-slice PBIs. Minting a standalone "build the audit store" PBI would be a horizontal split with no independent user value (red-flag anti-pattern). The append-only, same-transaction audit obligation (SR-AUD-001/002, SDD-CMP-008) rides on PBI-QUE-002 and PBI-SUB-002; the delivery-status ledger (SR-INT-003, SDD-DAT-007) rides on PBI-RMD-002.
- **Send window (ASSUMPTION-13 / OQ-7) and SMS consent-to-contact (ASSUMPTION-11 / OQ-5)** are un-ratified and un-owned upstream; both shape PBI-RMD-001's AC. If OQ-5 resolves to an explicit opt-in requirement (TCPA), that becomes a launch-gating P0 enabler and a new PBI splits out — flagged, not invented here.
- **Tasks stay on the tracker.** Below-story decomposition is created and owned on Azure Boards (`templates/task-template.md`), not mirrored here; this doc's hierarchy stops at PBI.
- **Seeding is one-way.** After this seeds Azure Boards (via `hooks/emit_backlog.py` from merged `main`), the board is the system of record for AC/status; this markdown keeps only the requirement-trace job and the resulting work-item IDs are written back into the RTM's Azure column.
